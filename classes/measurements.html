Measurements

Source: src/lagrangian/DSMC/macroscopicProperties/, src/lagrangian/DSMC/cellMeasurements/, src/lagrangian/DSMC/boundaryMeasurements/.

The measurement subsystem is the non-invasive observer of the simulation: it reads the simulator-particle distribution and writes diagnostics, but does not alter the particle population or the collision logic. Three tiers coexist:

  • per-cell primitive accumulators — cellMeasurements — hold the raw moments on which every volume field is built. They are updated on every DSMC step by the cloud’s calculateFields() routine;

  • per-patch primitive accumulators — boundaryMeasurements — hold the face-level impact statistics on every boundary patch;

  • configurable field models — under macroscopicProperties/derived/ — which the user switches on in system/fieldPropertiesDict to produce the desired output: volume fields (dsmcVolFields), bins (dsmcBinsMethod), zone averages (dsmcZone), rovibronic population distributions (dsmcRovibronicPopulation), face-flux integrals (dsmcMassFluxSurface), and velocity / speed histograms (dsmcVelocityDistributionZone, dsmcSpeedDistributionZone).

Each field model shares a common lifecycle: a constructor that reads its own sub-dictionary, createField() that allocates the storage, calculateField() invoked on every DSMC step, writeField() invoked at the output interval, and an updateProperties() hook that re-reads dictionary keys on the fly. The container class dsmcFieldProperties (macroscopicProperties/basic/dsmcFieldProperties/) iterates through the registered field models and orchestrates these calls. Field models are therefore trivially composable: one can attach any number of them to the same run, with different species filters, zones, and output cadences.

Design philosophy — on-the-fly accumulation

The measurement framework is built on two design choices inherited from the dsmcFoam+ lineage [White2013m] and preserved in the rovibronic extension:

  • On-the-fly accumulation. Raw particle properties — position, velocities, species type, internal state — are too voluminous to write out every time-step for post-processing. Instead, every measurement performs its sampling on the fly: at each DSMC step the relevant moments are accumulated in memory, and only the processed macroscopic quantities are written to disk at the user-specified write interval. This turns a memory-bound problem (writing \(O(10^{6\text{-}9})\) particles every step for \(O(10^{5\text{-}8})\) steps) into a compute-bound one.

  • Burn-in control. Every measurement model exposes a resetAtOutput switch together with a resetAtOutputUntilTime cut-off, so that the user can let the flow evolve to steady state and only then start accumulating statistics. The same switch allows long-window averaging in steady flows or continuous accumulation in ensemble-averaged unsteady flows.

  • Species filtering. Every model accepts a typeIds list; sampling is restricted to parcels whose species index matches. Leaving the list empty selects every species in the simulation. Multiple instances of the same model with different typeIds can therefore be attached to the same run to produce per-species diagnostics alongside the mixture-averaged ones.

  • Parallelism. Every sampler is fully parallelised. Per-cell and per-face accumulators live on the owning rank only; global reductions are performed at the write interval, so that the per-step overhead is strictly local. Sampling zones that span rank boundaries are handled by the reduction documented below.

All measurement models share these ingredients and are declared in the single system/fieldPropertiesDict file:

fields
(
    volFieldsN2 { fieldModel dsmcVolFields;            /* … */ }
    binsShock   { fieldModel dsmcBinsMethod;           /* … */ }
    zoneReactor { fieldModel dsmcZone;                 /* … */ }
    popO2       { fieldModel dsmcRovibronicPopulation; /* … */ }
    fluxInlet   { fieldModel dsmcMassFluxSurface;      /* … */ }
    velInflow   { fieldModel dsmcVelocityDistributionZone; /* … */ }
);

Parallel reduction

For a measurement volume \(\mathcal{Z}\) (a zone of cells, a bin, or the whole mesh) that may be split across \(n_{\mathrm{proc}}\) MPI ranks, the total number of simulator parcels in \(\mathcal{Z}\) is

\[N_{\mathcal{Z}} = \sum_{p=1}^{n_{\mathrm{proc}}} \sum_{c=1}^{n_{\mathrm{cells}}^{(p)}} N_c,\]

where \(N_c\) is the number of parcels in cell \(c\) on rank \(p\) and \(n_{\mathrm{cells}}^{(p)}\) is the number of \(\mathcal{Z}\)’s cells resident on that rank. The same reduction is applied to every accumulated moment — mass, momentum, kinetic energy, internal energy, higher-order tensors. Processor patches receive an additional \(0.5\) weighting on shared faces to avoid double-counting in face-flux measurements (see below).

Number-density normalisation

With simulator particles each representing \(F_N\) real molecules, the number density in a measurement volume is

\[n_{\mathcal{Z}} = \frac{F_N\,N_{\mathcal{Z}}} {\displaystyle\sum_{c \in \mathcal{Z}} V_c},\]

with \(V_c\) the mesh volume of cell \(c\). The division by the total volume of \(\mathcal{Z}\)’s cells — not just by \(V_{\mathrm{cell}}\) — is what makes the zone model an honest cell-averaged estimator independent of the zone’s cell count; the factor \(F_N\) is the nEquivalentParticles key of Kinetic foundations and the DSMC algorithm. Every derived field (mass density, velocity, stress, heat flux) inherits the same normalisation.

Statistical estimator

Every macroscopic field of interest is a moment of the parcel distribution that is accumulated in time, yielding (for a time-averaged measurement)

\[\overline{Q} = \frac{1}{N_{\mathrm{samples}}} \sum_{k=1}^{N_{\mathrm{samples}}} \frac{1}{N_{\mathcal{Z}}^{(k)}} \sum_{i \in \mathcal{Z}^{(k)}} q_i^{(k)},\]

where \(N_{\mathrm{samples}}\) is the number of DSMC steps in the current accumulation window and \(q_i\) is the per-parcel quantity (mass, momentum component, etc.). The statistical error on \(\overline{Q}\) goes as \(\sigma_Q / \sqrt{N_{\mathrm{samples}} N_{\mathcal{Z}}}\), so that the statistical quality of a measurement improves both with parcel density and with averaging time. The measureErrors switch on dsmcVolFields writes the per-cell \(\sigma_Q\) companion fields so that 95% confidence intervals can be reconstructed post-hoc.

Time-averaging controls

Every field model shares the same timeProperties sub-dictionary:

timeProperties
{
    timeOption          write;        // write  |  total  |  average
    resetAtOutput       on;           // on: flush accumulator each write
    resetAtOutputUntilTime  0;        // optional burn-in cut-off
}

  • timeOption write — accumulate between writes; reset after each write if resetAtOutput is on.

  • timeOption total — accumulate from \(t = 0\) to the current time (cumulative average).

  • timeOption average — accumulate from the end of the burn-in (resetAtOutputUntilTime) onwards.

The accumulator counter is the single knob that trades statistical resolution against temporal resolution: a longer window gives tighter error bars on each written field but averages over any unsteadiness that exists within the window.

cellMeasurements — the per-cell primitive accumulator

cellMeasurements holds, per cell and per species, the raw moments on which every downstream field is built. Quoting the actual member variables of the class (under src/lagrangian/DSMC/cellMeasurements/cellMeasurements.H):

  • \(N_s\) and \(\rho_s\) — counts and mass accumulators per species per cell;

  • the velocity moments \(\langle m v_x\rangle\), \(\langle m v_y\rangle\), \(\langle m v_z\rangle\), \(\langle m v^2\rangle\), and the full mass-weighted \(\langle m v_\alpha v_\beta\rangle\) tensor through the symmetric components muu, muv, muw, mvv, mvw, mww;

  • the translational heat-flux triple \(\langle m v^2 v_\alpha\rangle\) through mccu, mccv, mccw;

  • the internal-energy accumulator \(\langle E^\mathrm{int}\rangle\) per species, and its square for variance estimation;

  • the collision-count counters nColls_ and the mean collision separation collisionSeparation_.

These are incremented on every DSMC step by the cloud’s calculateFields() method and are consumed downstream by every field model below.

boundaryMeasurements — the per-patch primitive accumulator

On every parcel–face intersection, boundaryMeasurements increments:

  • rhoNBF_, rhoNIntBF_, rhoMBF_ — number and mass densities on the patch face (with and without the internal- state weighting);

  • momentumBF_ — the integrated momentum vector per face;

  • UCollectedBF_ — bulk velocity on the patch face;

  • linearKEBF_ — translational kinetic energy delivered to the wall;

  • internalEBF_ — rovibronic internal energy delivered to the wall, species by species;

  • heatFluxBF_ and qBF_ — heat-flux vector at the face;

  • frictionDragBF_ — friction drag vector.

These are written as boundary fields in the standard OpenFOAM output and are the ingredients for every wall-level analysis (surface heat flux, drag, catalytic chemistry). The rovibronic-aware internalEBF_ is the key addition over the stock dsmcFoam+ boundary instrumentation, because it carries the state-resolved internal energy delivered to the wall rather than a single continuous number.

Volume-field model — dsmcVolFields

Theoretical basis — three-dimensional method of cells

dsmcVolFields is conceptually the three-dimensional method of cells: at each DSMC step, every cell of the mesh is visited, the parcels it contains are sampled, and the accumulated moments are converted to macroscopic fields at the write interval. The spatial resolution is inherited from the mesh — as dense or as coarse as the mesh allows — and therefore defers to the sub-cell adaptation layer (Adaptation and controllers) for tightening the resolution locally without a re-mesh. Every macroscopic field of Macroscopic properties is built from the per-cell moments of \(N\), \(\rho\mathbf{v}\), \(\rho v^{2}\), \(\rho\mathbf{c}\mathbf{c}\), and the internal energy accumulated per species; the per-cell bisection inversion against the rovibronic partition function delivers the effective internal temperature.

The dsmcVolFields field model (macroscopicProperties/derived/combined/dsmcVolFields/) is the default production measurement. It produces the standard DSMC volume fields at the case’s writeInterval:

Bin model — dsmcBinsMethod

Theoretical basis — Method of Bins (MOB)

The Method of Bins, as described in White’s thesis [White2013m], splits a user-defined region of the domain into \(n_b\) bins and accumulates the spatial average of the desired properties inside each bin. The bins are independent of the computational mesh, so that the measurement resolution can be either finer or coarser than the mesh, and can follow a geometry (line, radial shell, angular sector) unrelated to the mesh topology. This is the tool of choice when the DSMC mesh has been designed for collision resolution (\(\lambda/3\)-sized cells) but the post-processing plots must be sampled on a different geometry — a one-dimensional line through a shock, a radial profile around an axisymmetric body, or a cylindrical cross-section through a nozzle.

For efficiency the MOB is merged with the Method of Zones (below): particles are looped over per-cell rather than per- particle, so that particles outside the measurement zone contribute zero cost. For a segmental bin stack between \(\mathbf{r}_s\) and \(\mathbf{r}_e\) with \(n_b\) uniformly-spaced bins of width

\[\Delta r = \frac{|\mathbf{r}_e - \mathbf{r}_s|}{n_b},\]

a particle at position \(\mathbf{r}_i\) is assigned the bin index

\[b_i = \left\lfloor \frac{(\mathbf{r}_i - \mathbf{r}_s)\cdot (\mathbf{r}_e - \mathbf{r}_s)} {\Delta r\,|\mathbf{r}_e - \mathbf{r}_s|} \right\rfloor,\]

projecting the particle’s displacement along the bin axis and normalising by the bin width. An analogous projection rule handles cylindrical, radial, and spherical bin geometries. Per- bin totals are then divided by the bin volume \(A_b\,\Delta r\) (area entry × bin width) to recover proper densities and intensive quantities.

Usage

The dsmcBinsMethod field model (macroscopicProperties/derived/combined/dsmcBinsMethod/) measures macroscopic properties in bins that are independent of the mesh cells. The returned values are bin averages. This is the tool of choice when the DSMC mesh has been designed for collision resolution (λ/3-sized cells), but the desired post-processing plots must be sampled on a different geometry — typically a one-dimensional line through a shock, a radial profile around an axisymmetric body, or a cylindrical cross section through a nozzle.

Bin model sub-classes

The geometry of the bins is chosen by a pluggable binModel sub-class. The available sub-classes, under src/lagrangian/DSMC/LEGACY/binModel/derived/, are:

Radial bins around an axis (absolute or normalised by the local body radius). The normalised variant is the standard tool for blunt-body re-entry post-processing.

sphericalBins

Concentric spherical shells around a point; useful for expansion plume analysis.

angularBins

Angular bins at fixed radius; for scattering-direction histograms.

cylinderBin

Annular shells in a cylindrical coordinate frame.

multiZoneBins

A composition: the same bin geometry repeated, once per entry of a cellZoneList; used when several sub-domains of the mesh need independent bin stacks with the same resolution.

Each of these lives under src/lagrangian/DSMC/LEGACY/binModel/derived/<sub-class>/ and exposes its own <subclass>Properties block.

Fields measured in each bin

The bin model accumulates, per bin, the same primitive moments as the per-cell accumulators: number and mass density, mean velocity, the full peculiar-velocity moment tensor (muu, muv, muw, mvv, mvw, mww), and the internal-energy moments. From these, the output writes (per bin, at every write step):

  • N, M, rhoN, rhoM — population and density measurements;

  • UMean and scalarUMean — bulk velocity;

  • translationalTemperature, internalTemperature — bisection-inverted temperatures;

  • pressureTensor and its deviatoric shearStressTensor, together with the scalar scalarPressure;

  • heatFlux, split into heatFluxTranslational and heatFluxInternal contributions;

  • meanFreePath and Mach per bin;

  • Cv, Cp, gamma — heat capacities from the bin-level internal partition function.

Configuration (verbatim from the class header):

fieldModel   dsmcBinsMethod;
timeProperties { timeOption write; resetAtOutput on; }
dsmcBinsMethodProperties
{
    field                   N2;
    zone                    channelRegion;    // optional cellZone filter
    typeIds                 (N2);
    averagingAcrossManyRuns no;
    binModel                uniformBins;
    uniformBinsProperties
    {
        startPoint  (0 0 0);
        endPoint    (2e-6 0 0);
        nBins       60;
        area        2.64e-14;   // cross-section for density
    }
}

The output is written to <case>/fieldMeasurements/<field>/ as plain-text tables, one row per bin per write step, and is therefore directly usable in Python / gnuplot / Matlab pipelines.

Zone model — dsmcZone

Theoretical basis — Method of Zones (MOZ)

The Method of Zones, as introduced in White’s thesis [White2013m], is the conceptually simplest sampling technique: a measurement zone \(\mathcal{Z}\) is defined as a collection of connected mesh cells — which may span any shape, size, or location on the mesh — and all sampled quantities are spatial averages over \(\mathcal{Z}\). Because the zone is one collection of cells, the method cannot resolve spatial gradients inside the zone; the value returned is a single scalar, vector, or tensor per write step. This is the natural tool when the flow is spatially homogeneous inside \(\mathcal{Z}\), or when only an integral quantity (a reactor-averaged rate, a thrust, a heat-balance budget) is needed.

The efficiency of the MOZ relies on OpenFOAM’s cell occupancy index: because the cloud already knows which parcels live in which cell after the movement stage, there is no need to loop over all parcels and test membership. The sampler loops only over the cells of \(\mathcal{Z}\), and over the parcels that each such cell holds. The number density in the zone, accounting for the possibility that \(\mathcal{Z}\) is split across MPI ranks, is

\[n_{\mathcal{Z}} = \frac{F_N\,N_{\mathcal{Z}}} {\displaystyle\sum_{c\in\mathcal{Z}} V_c}, \qquad N_{\mathcal{Z}} = \sum_{p=1}^{n_{\mathrm{proc}}} \sum_{c=1}^{n_{\mathrm{cells}}^{(p, \mathcal{Z})}} N_c,\]

which is the same pair of formulas given in the opening section. Higher moments (momentum, stress, heat flux, internal energy) follow the same reduction.

Rovibronic extension

The rovibronic variant of dsmcZone differs from the dsmcFoam+ original in its internal-temperature diagnostic: rather than computing \(T_\mathrm{int}\) from a species-dependent harmonic-oscillator closure (which is inconsistent across species in a mixture), it accumulates the raw internal energy per species and inverts \(\langle E_\mathrm{int}^{(s)}\rangle(T_{\mathrm{int}}^{(s)})\) by Newton-Raphson against the pre-tabulated rovibronic partition function (Rovibronic level structure). Mixture-averaged temperatures are built from the per-species effective temperatures weighted by population.

Usage

The dsmcZone field model (macroscopicProperties/derived/combined/dsmcZone/) returns the volume-averaged macroscopic properties over a named cellZone. Works in parallel. Rovibronic version: it accumulates the internal energy directly from the parcels and inverts \(T_\mathrm{int}\) through a Newton-Raphson loop against the mode-specific partition function.

Output fields

For each write, the zone model writes a single-row record containing the zone-averaged:

  • number and mass densities;

  • translational and effective internal temperatures;

  • pressure tensor and its scalar trace;

  • mean velocity;

  • heat-flux and friction-drag vectors;

  • shear-stress tensor;

  • per-species internal energy;

  • collision-rate statistics (if measureCollisionRateOnly is true, only this last group is written, which is the cheapest possible measurement mode).

Configuration:

fieldModel  dsmcZone;
timeProperties { timeOption write; resetAtOutput off; }
dsmcZoneProperties
{
    zone                      combustionChamberZone;
    field                     O2;           // output tag
    typeIds                   (O2);
    measureCollisionRateOnly  false;
}

The zone model is the natural tool for 0-D reference quantities — heat-bath relaxation curves, reactor-averaged production rates, single-cell time histories — because one cellZone of the mesh becomes a single-value time series in the output file. Combining several dsmcZone instances with different species filters on the same zone gives per-species time histories alongside the mixture-averaged values.

Rovibronic population — dsmcRovibronicPopulation

Theoretical basis — state-resolved distribution function

Where the scalar macroscopic fields of Macroscopic properties summarise the internal state of the gas through effective temperatures that invert the first moment of the population distribution, the state-resolved population diagnostic is the distribution itself. For a species \(s\) in a sampling zone, we define

\[P^{(s)}(n, v, J) = \frac{1}{N^{(s)}_{\mathcal{Z}}} \sum_{i\,\in\,\mathcal{Z}^{(s)}} \delta_{(n_i, v_i, J_i),\,(n, v, J)},\]

the fraction of parcels in \(\mathcal{Z}\) whose quantum state indices match \((n, v, J)\). Summation over the level space returns unity by construction,

\[\sum_{n, v, J} P^{(s)}(n, v, J) = 1,\]

and the average internal energy is

\[\langle E_\mathrm{int}^{(s)}\rangle = \sum_{n, v, J} E^{(s)}(n, v, J)\,P^{(s)}(n, v, J).\]

The diagnostic writes the four-index array \((c, n, v, J) \mapsto P(c; n, v, J)\) (per cell of the sampling zone) together with the energy and degeneracy look-up tables, so that plots of the form “log \(P(v) / g(v)\) versus \(E(v)\)” reveal the departure from Boltzmann directly: a Boltzmann distribution appears as a straight line whose slope is \(-1/(k_B T)\), and any curvature is a non-Boltzmann signature.

This is the only observable that directly reflects the central promise of the rovibronic extension.

Usage

The dsmcRovibronicPopulation field model (macroscopicProperties/derived/otherFields/dsmcRovibronicPopulation/) is the diagnostic that directly exercises the central promise of the rovibronic extension. It accumulates, per cell and per rovibronic level, the population of a selected species:

\[P(\mathrm{cell};\,n, v, J) = \frac{1}{N}\sum_{i\in\mathrm{cell}} \delta_{(n_i, v_i, J_i),\,(n, v, J)},\]

so that \(\sum_{n, v, J} P(\mathrm{cell}; n, v, J) = 1\). The output is a four-dimensional array indexed by \((\mathrm{cell}, n, v, J)\) together with the corresponding energy and degeneracy tables, and is the only measurement that gives access to the non-Boltzmann shape of the populated levels.

Configuration:

fieldModel  dsmcRovibronicPopulation;
dsmcRovibronicPopulationProperties
{
    fieldName   N2_population;
    zoneName    shockRegion;
    typeId      N2;          // one species per instance
}

Typical plots built from this output:

  • the vibrational population \(P(v)\) at fixed electronic state \(n\), with the Boltzmann distribution at \(T_t\) overlaid as a reference — the single most direct diagnostic of vibrational non-equilibrium;

  • the rotational population \(P(J)\) at fixed \((n, v)\), used to test the rotational-vibrational equilibrium assumption;

  • the electronic population \(P(n)\) as a time series in shock-heated flows, showing how the excited electronic manifolds are progressively populated.

Face-flux measurement — dsmcMassFluxSurface

Theoretical basis — fluxal measurements on faces

OpenFOAM tags every interior and boundary face with an owner cell \(P\) and a neighbour cell \(Q\), and the face normal \(\hat{\mathbf{n}}_f\) is defined as pointing from \(P\) to \(Q\). A parcel \(i\) crossing face \(f\) with velocity \(\mathbf{c}_i\) contributes to the face flux with sign

\[\begin{split}\mathrm{sgn}(\mathbf{c}_i \cdot \hat{\mathbf{n}}_f) = \begin{cases} +1, & \mathbf{c}_i \cdot \hat{\mathbf{n}}_f > 0,\\ \;\;0, & \mathbf{c}_i \cdot \hat{\mathbf{n}}_f = 0,\\ -1, & \mathbf{c}_i \cdot \hat{\mathbf{n}}_f < 0, \end{cases}\end{split}\]

so that an outgoing parcel (positive with respect to the face normal) contributes \(+m_i\), an incoming parcel contributes \(-m_i\), and a parcel moving exactly tangentially contributes nothing. The OpenFOAM particle-tracking algorithm flags every such face crossing during a time-step, so no extra geometric search is required.

Summing over the time-averaging window, the mean mass flow rate through a face \(f\) of area \(A_f\) is

\[\langle \dot{m}_f\rangle = \frac{1}{t_{\mathrm{av}}}\, \sum_{i\,\in\,\Delta N_f(t \to t_n)} m_i\,\mathrm{sgn}(\mathbf{c}_i\cdot\hat{\mathbf{n}}_f),\]

with \(t_{\mathrm{av}}\) the averaging duration and \(\Delta N_f(t \to t_n)\) the total number of parcels that cross \(f\) during the interval. The corresponding mass flux per unit area is \(\langle\dot{m}_f\rangle / A_f\). The sampler accumulates this on a per-face basis and writes totals for the union of faces in a user-named faceZone — so that the flux through an inlet, through a periodic surrogate face, or across a plane cutting the domain can be measured directly from the simulation without post-processing.

Parallel treatment

Faces that lie on a processor patch (a shared boundary between two MPI ranks) are weighted by \(\tfrac{1}{2}\) when accumulated on each side, so that after global reduction the total is counted exactly once. Without this factor, processor- local flux sums would be inflated by a factor of two on every parallel-patch face.

Usage

dsmcMassFluxSurface (macroscopicProperties/derived/massFlux/dsmcMassFluxSurface/) integrates mass, number, momentum, and energy fluxes through a user-defined faceZone. On processor patches the contribution of a face is weighted by 0.5 so that the total is correct on reconstruction after a parallel run. The sub-dictionary takes:

fieldModel  dsmcMassFluxSurface;
dsmcMassFluxSurfaceProperties
{
    field            inletFlux;
    typeIds          (N2 O2);
    faceZone         inletFace;
    fluxDirection    (1 0 0);    // sign convention
}

Useful for inlet / outlet monitoring, mass conservation checks, and for extracting mass-flow boundary-condition data from a reference run to feed an dsmcMassFlowRateInlet in a downstream case.

Velocity and speed distributions

Theoretical basis — translational distribution functions

The two velocity-distribution tools accumulate histograms of, respectively, the Cartesian velocity components \(P^{(s)}(v_\alpha)\) for \(\alpha \in \{x, y, z\}\) and the speed magnitude \(P^{(s)}(|\mathbf{c}|)\), each over a named cellZone. In a gas in translational equilibrium, the Maxwell-Boltzmann distributions are

\[P_\mathrm{MB}(v_\alpha) = \left(\frac{m_s}{2\pi k_B T}\right)^{1/2} \exp\!\left[-\frac{m_s\,(v_\alpha - \bar{v}_\alpha)^{2}} {2 k_B T}\right],\]
\[P_\mathrm{MB}(|\mathbf{c}|) = 4\pi\,|\mathbf{c}|^{2} \left(\frac{m_s}{2\pi k_B T}\right)^{3/2} \exp\!\left[-\frac{m_s\,|\mathbf{c}|^{2}} {2 k_B T}\right].\]

Departures of the sampled histogram from these references are the primary diagnostic of translational non-equilibrium — specifically, the bimodal speed distribution that characterises the interior of a strong shock, or the elongated tail of a rapid expansion. Deviations from \(P_\mathrm{MB}(|\mathbf{c}|)\) in the exit region of a micro-channel flow were the original motivation for the MOZ-based diagnostic in White’s thesis [White2013m].

Usage

dsmcVelocityDistributionZone and dsmcSpeedDistributionZone (macroscopicProperties/derived/velocity/) accumulate histograms of, respectively, the velocity-component and speed-magnitude distribution of the parcels inside a named zone. The histogram bin width is set through the binWidth key (in m/s).

These are the primary tools to confirm that the sampled velocity distribution remains Maxwellian under non-equilibrium, or to quantify the departure when it does not. Typical use:

fieldModel  dsmcSpeedDistributionZone;
dsmcSpeedDistributionZoneProperties
{
    field       inflowSpeed;
    zoneName    inflowZone;
    typeIds     (N2);
    binWidth    50;        // m/s bins
}

Orchestration and lifecycle

Every field model in the present section inherits from dsmcField (macroscopicProperties/basic/dsmcField/) and is instantiated by the container class dsmcFieldProperties (macroscopicProperties/basic/dsmcFieldProperties/) which reads the top-level system/fieldPropertiesDict:

fields
(
    fieldA { fieldModel dsmcVolFields;              /* … */ }
    fieldB { fieldModel dsmcBinsMethod;             /* … */ }
    fieldC { fieldModel dsmcZone;                   /* … */ }
    fieldD { fieldModel dsmcRovibronicPopulation;   /* … */ }
    fieldE { fieldModel dsmcMassFluxSurface;        /* … */ }
);

On every DSMC step the cloud invokes dsmcFieldProperties::calculateField(), which dispatches to each registered field model in sequence; on every write step it invokes writeField() similarly. Field models are therefore trivially stacked: attach as many as are useful, with different species filters, zones, and time options. A zone-averaged dsmcZone combined with a full-volume dsmcVolFields, a bin-averaged dsmcBinsMethod across a shock, and a state-resolved dsmcRovibronicPopulation in the shock layer is a fairly standard combination for a re-entry simulation.
