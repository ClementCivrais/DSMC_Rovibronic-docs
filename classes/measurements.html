Measurements

Source: src/lagrangian/DSMC/macroscopicProperties/, src/lagrangian/DSMC/cellMeasurements/, src/lagrangian/DSMC/boundaryMeasurements/.

The measurement subsystem is the non-invasive observer of the simulation: it reads the simulator-particle distribution and writes diagnostics, but does not alter the particle population or the collision logic. Three tiers coexist:

  • per-cell primitive accumulators — cellMeasurements — hold the raw moments on which every volume field is built. They are updated on every DSMC step by the cloud’s calculateFields() routine;

  • per-patch primitive accumulators — boundaryMeasurements — hold the face-level impact statistics on every boundary patch;

  • configurable field models — under macroscopicProperties/derived/ — which the user switches on in system/fieldPropertiesDict to produce the desired output: volume fields (dsmcVolFields), bins (dsmcBinsMethod), zone averages (dsmcZone), rovibronic population distributions (dsmcRovibronicPopulation), face-flux integrals (dsmcMassFluxSurface), and velocity / speed histograms (dsmcVelocityDistributionZone, dsmcSpeedDistributionZone).

Each field model shares a common lifecycle: a constructor that reads its own sub-dictionary, createField() that allocates the storage, calculateField() invoked on every DSMC step, writeField() invoked at the output interval, and an updateProperties() hook that re-reads dictionary keys on the fly. The container class dsmcFieldProperties (macroscopicProperties/basic/dsmcFieldProperties/) iterates through the registered field models and orchestrates these calls. Field models are therefore trivially composable: one can attach any number of them to the same run, with different species filters, zones, and output cadences.

Time-averaging controls

Every field model shares the same timeProperties sub-dictionary:

timeProperties
{
    timeOption          write;        // write  |  total  |  average
    resetAtOutput       on;           // on: flush accumulator each write
    resetAtOutputUntilTime  0;        // optional burn-in cut-off
}

  • timeOption write — accumulate between writes; reset after each write if resetAtOutput is on.

  • timeOption total — accumulate from \(t = 0\) to the current time (cumulative average).

  • timeOption average — accumulate from the end of the burn-in (resetAtOutputUntilTime) onwards.

The accumulator counter is the single knob that trades statistical resolution against temporal resolution: a longer window gives tighter error bars on each written field but averages over any unsteadiness that exists within the window.

cellMeasurements — the per-cell primitive accumulator

cellMeasurements holds, per cell and per species, the raw moments on which every downstream field is built. Quoting the actual member variables of the class (under src/lagrangian/DSMC/cellMeasurements/cellMeasurements.H):

  • \(N_s\) and \(\rho_s\) — counts and mass accumulators per species per cell;

  • the velocity moments \(\langle m v_x\rangle\), \(\langle m v_y\rangle\), \(\langle m v_z\rangle\), \(\langle m v^2\rangle\), and the full mass-weighted \(\langle m v_\alpha v_\beta\rangle\) tensor through the symmetric components muu, muv, muw, mvv, mvw, mww;

  • the translational heat-flux triple \(\langle m v^2 v_\alpha\rangle\) through mccu, mccv, mccw;

  • the internal-energy accumulator \(\langle E^\mathrm{int}\rangle\) per species, and its square for variance estimation;

  • the collision-count counters nColls_ and the mean collision separation collisionSeparation_.

These are incremented on every DSMC step by the cloud’s calculateFields() method and are consumed downstream by every field model below.

boundaryMeasurements — the per-patch primitive accumulator

On every parcel–face intersection, boundaryMeasurements increments:

  • rhoNBF_, rhoNIntBF_, rhoMBF_ — number and mass densities on the patch face (with and without the internal- state weighting);

  • momentumBF_ — the integrated momentum vector per face;

  • UCollectedBF_ — bulk velocity on the patch face;

  • linearKEBF_ — translational kinetic energy delivered to the wall;

  • internalEBF_ — rovibronic internal energy delivered to the wall, species by species;

  • heatFluxBF_ and qBF_ — heat-flux vector at the face;

  • frictionDragBF_ — friction drag vector.

These are written as boundary fields in the standard OpenFOAM output and are the ingredients for every wall-level analysis (surface heat flux, drag, catalytic chemistry). The rovibronic-aware internalEBF_ is the key addition over the stock dsmcFoam+ boundary instrumentation, because it carries the state-resolved internal energy delivered to the wall rather than a single continuous number.

Volume-field model — dsmcVolFields

The dsmcVolFields field model (macroscopicProperties/derived/combined/dsmcVolFields/) is the default production measurement. It produces the standard DSMC volume fields at the case’s writeInterval:

Configuration keys (dsmcVolFieldsProperties):

fieldModel    dsmcVolFields;
timeProperties { timeOption write; resetAtOutput on; }
dsmcVolFieldsProperties
{
    field                        N2O2;           // file-name tag for this instance
    typeIds                      (N2 O2);        // species filter
    averagingAcrossManyRuns      false;
    measureMeanFreePath          true;
    measureErrors                true;           // writes std-dev companions
    densityOnly                  false;          // if true, only rhoN / rhoM
    measureHeatFluxShearStress   true;           // q, τ, P
}

Switching measureErrors to true writes a companion <field>_errorEstimate volume field alongside each observable, populated with the per-cell standard deviation \(\sigma_Q / \sqrt{N}\); this is the quantity needed to construct confidence intervals on the macroscopic output (see Kinetic foundations and the DSMC algorithm). Keeping measureErrors on during production runs has a small overhead and is strongly recommended.

Bin model — dsmcBinsMethod

The dsmcBinsMethod field model (macroscopicProperties/derived/combined/dsmcBinsMethod/) measures macroscopic properties in bins that are independent of the mesh cells. The returned values are bin averages. This is the tool of choice when the DSMC mesh has been designed for collision resolution (λ/3-sized cells), but the desired post-processing plots must be sampled on a different geometry — typically a one-dimensional line through a shock, a radial profile around an axisymmetric body, or a cylindrical cross section through a nozzle.

Bin model sub-classes

The geometry of the bins is chosen by a pluggable binModel sub-class. The available sub-classes, under src/lagrangian/DSMC/LEGACY/binModel/derived/, are:

binModel

Geometry

uniformBins

A uniformly-spaced 1-D stack of slabs between two points; configured by startPoint, endPoint, nBins, and area (cross-sectional area normal to the stack, used to convert bin volume into the correct density normalisation).

simpleBins

Axis-aligned uniform 1-D bins along \(x\), \(y\), or \(z\); specified by the bin-size vector and origin.

cuboidBin / cuboidBinII

Rectangular parallelepipeds, with one or two degrees of freedom; useful for 2-D slicing of 3-D domains.

radialBins / radialBinsNormalised

Radial bins around an axis (absolute or normalised by the local body radius). The normalised variant is the standard tool for blunt-body re-entry post-processing.

sphericalBins

Concentric spherical shells around a point; useful for expansion plume analysis.

angularBins

Angular bins at fixed radius; for scattering-direction histograms.

cylinderBin

Annular shells in a cylindrical coordinate frame.

multiZoneBins

A composition: the same bin geometry repeated, once per entry of a cellZoneList; used when several sub-domains of the mesh need independent bin stacks with the same resolution.

Each of these lives under src/lagrangian/DSMC/LEGACY/binModel/derived/<sub-class>/ and exposes its own <subclass>Properties block.

Fields measured in each bin

The bin model accumulates, per bin, the same primitive moments as the per-cell accumulators: number and mass density, mean velocity, the full peculiar-velocity moment tensor (muu, muv, muw, mvv, mvw, mww), and the internal-energy moments. From these, the output writes (per bin, at every write step):

  • N, M, rhoN, rhoM — population and density measurements;

  • UMean and scalarUMean — bulk velocity;

  • translationalTemperature, internalTemperature — bisection-inverted temperatures;

  • pressureTensor and its deviatoric shearStressTensor, together with the scalar scalarPressure;

  • heatFlux, split into heatFluxTranslational and heatFluxInternal contributions;

  • meanFreePath and Mach per bin;

  • Cv, Cp, gamma — heat capacities from the bin-level internal partition function.

Configuration (verbatim from the class header):

fieldModel   dsmcBinsMethod;
timeProperties { timeOption write; resetAtOutput on; }
dsmcBinsMethodProperties
{
    field                   N2;
    zone                    channelRegion;    // optional cellZone filter
    typeIds                 (N2);
    averagingAcrossManyRuns no;
    binModel                uniformBins;
    uniformBinsProperties
    {
        startPoint  (0 0 0);
        endPoint    (2e-6 0 0);
        nBins       60;
        area        2.64e-14;   // cross-section for density
    }
}

The output is written to <case>/fieldMeasurements/<field>/ as plain-text tables, one row per bin per write step, and is therefore directly usable in Python / gnuplot / Matlab pipelines.

Zone model — dsmcZone

The dsmcZone field model (macroscopicProperties/derived/combined/dsmcZone/) returns the volume-averaged macroscopic properties over a named cellZone. Works in parallel. Rovibronic version: it accumulates the internal energy directly from the parcels and inverts \(T_\mathrm{int}\) through a Newton-Raphson loop against the mode-specific partition function.

Output fields

For each write, the zone model writes a single-row record containing the zone-averaged:

  • number and mass densities;

  • translational and effective internal temperatures;

  • pressure tensor and its scalar trace;

  • mean velocity;

  • heat-flux and friction-drag vectors;

  • shear-stress tensor;

  • per-species internal energy;

  • collision-rate statistics (if measureCollisionRateOnly is true, only this last group is written, which is the cheapest possible measurement mode).

Configuration:

fieldModel  dsmcZone;
timeProperties { timeOption write; resetAtOutput off; }
dsmcZoneProperties
{
    zone                      combustionChamberZone;
    field                     O2;           // output tag
    typeIds                   (O2);
    measureCollisionRateOnly  false;
}

The zone model is the natural tool for 0-D reference quantities — heat-bath relaxation curves, reactor-averaged production rates, single-cell time histories — because one cellZone of the mesh becomes a single-value time series in the output file. Combining several dsmcZone instances with different species filters on the same zone gives per-species time histories alongside the mixture-averaged values.

Rovibronic population — dsmcRovibronicPopulation

The dsmcRovibronicPopulation field model (macroscopicProperties/derived/otherFields/dsmcRovibronicPopulation/) is the diagnostic that directly exercises the central promise of the rovibronic extension. It accumulates, per cell and per rovibronic level, the population of a selected species:

\[P(\mathrm{cell};\,n, v, J) = \frac{1}{N}\sum_{i\in\mathrm{cell}} \delta_{(n_i, v_i, J_i),\,(n, v, J)},\]

so that \(\sum_{n, v, J} P(\mathrm{cell}; n, v, J) = 1\). The output is a four-dimensional array indexed by \((\mathrm{cell}, n, v, J)\) together with the corresponding energy and degeneracy tables, and is the only measurement that gives access to the non-Boltzmann shape of the populated levels.

Configuration:

fieldModel  dsmcRovibronicPopulation;
dsmcRovibronicPopulationProperties
{
    fieldName   N2_population;
    zoneName    shockRegion;
    typeId      N2;          // one species per instance
}

Typical plots built from this output:

  • the vibrational population \(P(v)\) at fixed electronic state \(n\), with the Boltzmann distribution at \(T_t\) overlaid as a reference — the single most direct diagnostic of vibrational non-equilibrium;

  • the rotational population \(P(J)\) at fixed \((n, v)\), used to test the rotational-vibrational equilibrium assumption;

  • the electronic population \(P(n)\) as a time series in shock-heated flows, showing how the excited electronic manifolds are progressively populated.

Face-flux measurement — dsmcMassFluxSurface

dsmcMassFluxSurface (macroscopicProperties/derived/massFlux/dsmcMassFluxSurface/) integrates mass, number, momentum, and energy fluxes through a user-defined faceZone. On processor patches the contribution of a face is weighted by 0.5 so that the total is correct on reconstruction after a parallel run. The sub-dictionary takes:

fieldModel  dsmcMassFluxSurface;
dsmcMassFluxSurfaceProperties
{
    field            inletFlux;
    typeIds          (N2 O2);
    faceZone         inletFace;
    fluxDirection    (1 0 0);    // sign convention
}

Useful for inlet / outlet monitoring, mass conservation checks, and for extracting mass-flow boundary-condition data from a reference run to feed an dsmcMassFlowRateInlet in a downstream case.

Velocity and speed distributions

dsmcVelocityDistributionZone and dsmcSpeedDistributionZone (macroscopicProperties/derived/velocity/) accumulate histograms of, respectively, the velocity-component and speed-magnitude distribution of the parcels inside a named zone. The histogram bin width is set through the binWidth key (in m/s).

These are the primary tools to confirm that the sampled velocity distribution remains Maxwellian under non-equilibrium, or to quantify the departure when it does not. Typical use:

fieldModel  dsmcSpeedDistributionZone;
dsmcSpeedDistributionZoneProperties
{
    field       inflowSpeed;
    zoneName    inflowZone;
    typeIds     (N2);
    binWidth    50;        // m/s bins
}

Orchestration and lifecycle

Every field model in the present section inherits from dsmcField (macroscopicProperties/basic/dsmcField/) and is instantiated by the container class dsmcFieldProperties (macroscopicProperties/basic/dsmcFieldProperties/) which reads the top-level system/fieldPropertiesDict:

fields
(
    fieldA { fieldModel dsmcVolFields;              /* … */ }
    fieldB { fieldModel dsmcBinsMethod;             /* … */ }
    fieldC { fieldModel dsmcZone;                   /* … */ }
    fieldD { fieldModel dsmcRovibronicPopulation;   /* … */ }
    fieldE { fieldModel dsmcMassFluxSurface;        /* … */ }
);

On every DSMC step the cloud invokes dsmcFieldProperties::calculateField(), which dispatches to each registered field model in sequence; on every write step it invokes writeField() similarly. Field models are therefore trivially stacked: attach as many as are useful, with different species filters, zones, and time options. A zone-averaged dsmcZone combined with a full-volume dsmcVolFields, a bin-averaged dsmcBinsMethod across a shock, and a state-resolved dsmcRovibronicPopulation in the shock layer is a fairly standard combination for a re-entry simulation.
